# PROMPT: Cover Letter DOCX Formatting Standard — Winnow

## Purpose
This prompt defines the **exact formatting, fonts, styles, spacing, and margins** that ALL cover letters generated by Winnow must follow. It was extracted from the approved reference file `Reformatted_Cover_Letter.docx`. Any code that generates cover letter DOCX files MUST comply with these specs.

---

## Triggers — When to Use This Prompt
- Building or modifying the cover letter generation feature (Stage 4 / C_s)
- Creating DOCX cover letters from the tailoring pipeline
- Debugging or QA-ing cover letter output formatting
- Any PR that touches cover letter DOCX generation code

## Fonts

| Element         | Font Family  | Size     | Weight | Color              |
|----------------|-------------|----------|--------|--------------------|
| **Body text**   | Cambria (theme: minorHAnsi) | **11pt** (sz: 22 half-points) | Normal | Auto (black)       |
| **Name / Heading 1** | Calibri (theme: majorHAnsi) | **14pt** (sz: 28 half-points) | **Bold** | **#365F91** (accent1, themeShade BF) |

> **Important:** The document uses Word theme fonts. `minorHAnsi` → **Cambria**, `majorHAnsi` → **Calibri**. When generating with `docx-js`, set these fonts explicitly since theme resolution won't happen automatically.

## Paragraph Styles & Spacing

### Document Defaults (inherited by all paragraphs unless overridden)
| Property         | Value                        |
|-----------------|------------------------------|
| Font             | Cambria 11pt                 |
| After spacing    | **200** twips (≈ 10pt)       |
| Line spacing     | **276** twips (multiple: ~1.15 lines) |
| Line rule        | Auto                         |

### Heading 1 (Name line only)
| Property         | Value                        |
|-----------------|------------------------------|
| Font             | Calibri 14pt Bold            |
| Color            | #365F91                      |
| Before spacing   | **0** (overridden from default 480) |
| After spacing    | **0** (overridden from default 200) |
| Keep with next   | Yes                          |
| Keep lines       | Yes                          |

### Body Paragraphs (contact info, date, address, salutation, body text, closing)
| Property         | Value                        |
|-----------------|------------------------------|
| Font             | Cambria 11pt                 |
| After spacing    | **0** (overridden from default 200) |
| Line spacing     | Inherited default (276 / 1.15) |

### Blank Separator Lines
Empty paragraphs with **no spacing override** — they inherit the document default `after: 200`, producing a visual gap between sections.

## Document Structure (Top to Bottom)

The cover letter follows this exact block sequence. Each numbered item is a paragraph:

```
1.  [Heading1, spacing before=0] Full Name
2.  [Body, after=0]              City, ST • Phone • Email • LinkedIn
3.  [Body, after=0]              (empty line — blank paragraph, default spacing)
4.  [Body, after=0]              Date (e.g., February 8, 2026)
5.  (empty paragraph — default spacing = visual gap)
6.  [Body, after=0]              Hiring Manager (or contact name)
7.  [Body, after=0]              Company Name
8.  [Body, after=0]              [Address Line 1]
9.  [Body, after=0]              [City, State ZIP]
10. (empty paragraph — default spacing = visual gap)
11. [Body, after=0]              Dear Hiring Manager,
12. (empty paragraph — visual gap)
13. [Body, after=0]              Opening paragraph
14. (empty paragraph — visual gap)
15. [Body, after=0]              Body paragraph 2
16. (empty paragraph — visual gap)
17. [Body, after=0]              Body paragraph 3
18. (empty paragraph — visual gap)
19. [Body, after=0]              Body paragraph 4 (optional)
20. (empty paragraph — visual gap)
21. [Body, after=0]              Closing paragraph
22. (empty paragraph — visual gap)
23. [Body, default spacing]      Sincerely,
24. [Body, default spacing]      Full Name
```

> **Key pattern:** Content paragraphs use `spacing after="0"`. Blank paragraphs between sections inherit the default after=200 to create visual separation. This is how the document avoids double-spacing while keeping clean gaps between blocks.

## docx-js Implementation Reference

When generating cover letters with `docx-js` (Node.js `docx` package), use this configuration:

```javascript
const { Document, Packer, Paragraph, TextRun, HeadingLevel, AlignmentType } = require('docx');

const coverLetterDoc = new Document({
  styles: {
    default: {
      document: {
        run: {
          font: "Cambria",
          size: 22,          // 11pt in half-points
        },
        paragraph: {
          spacing: {
            after: 200,      // default after spacing
            line: 276,       // ~1.15 line spacing
          },
        },
      },
    },
    paragraphStyles: [
      {
        id: "Heading1",
        name: "Heading 1",
        basedOn: "Normal",
        next: "Normal",
        quickFormat: true,
        run: {
          font: "Calibri",
          size: 28,          // 14pt
          bold: true,
          color: "365F91",
        },
        paragraph: {
          spacing: { before: 0, after: 0 },
          keepNext: true,
          keepLines: true,
          outlineLevel: 0,
        },
      },
    ],
  },
  sections: [{
    properties: {
      page: {
        size: {
          width: 12240,      // 8.5" US Letter
          height: 15840,     // 11" US Letter
        },
        margin: {
          top: 1440,         // 1"
          bottom: 1440,      // 1"
          left: 1800,        // 1.25"
          right: 1800,       // 1.25"
          header: 720,       // 0.5"
          footer: 720,       // 0.5"
          gutter: 0,
        },
      },
    },
    children: [
      // 1. Full Name (Heading 1)
      new Paragraph({
        heading: HeadingLevel.HEADING_1,
        spacing: { before: 0 },
        children: [new TextRun("{{full_name}}")],
      }),

      // 2. Contact line
      new Paragraph({
        spacing: { after: 0 },
        children: [new TextRun("{{city}}, {{state}} • {{phone}} • {{email}} • {{linkedin}}")],
      }),

      // 3. Blank separator
      new Paragraph({ spacing: { after: 0 } }),

      // 4. Date
      new Paragraph({
        spacing: { after: 0 },
        children: [new TextRun("{{date}}")],
      }),

      // 5. Blank separator
      new Paragraph({}),

      // 6-9. Recipient address block (each line after=0)
      new Paragraph({ spacing: { after: 0 }, children: [new TextRun("{{recipient_name}}")] }),
      new Paragraph({ spacing: { after: 0 }, children: [new TextRun("{{company_name}}")] }),
      new Paragraph({ spacing: { after: 0 }, children: [new TextRun("{{address_line_1}}")] }),
      new Paragraph({ spacing: { after: 0 }, children: [new TextRun("{{city_state_zip}}")] }),

      // 10. Blank separator
      new Paragraph({}),

      // 11. Salutation
      new Paragraph({
        spacing: { after: 0 },
        children: [new TextRun("Dear {{recipient_name}},")],
      }),

      // 12. Blank separator
      new Paragraph({}),

      // 13. Opening paragraph
      new Paragraph({
        spacing: { after: 0 },
        children: [new TextRun("{{opening_paragraph}}")],
      }),

      // Repeat pattern: blank paragraph → body paragraph (after=0)
      // ... for each body paragraph ...

      // Final: blank → "Sincerely," → "Full Name" (both use default spacing)
      new Paragraph({}),
      new Paragraph({ children: [new TextRun("Sincerely,")] }),
      new Paragraph({ children: [new TextRun("{{full_name}}")] }),
    ],
  }],
});
```

## Formatting Rules Summary

1. **Page:** US Letter, 1" top/bottom margins, 1.25" left/right margins
2. **Body font:** Cambria 11pt, black, 1.15 line spacing
3. **Name heading:** Calibri 14pt Bold, color #365F91, no spacing before/after
4. **Content paragraphs:** after spacing = 0
5. **Section gaps:** Created by empty paragraphs inheriting default after=200
6. **Smart quotes:** Use `\u2019` (right single quote / apostrophe) not straight quotes
7. **No bullet points or numbered lists** in cover letters
8. **Single column, no headers/footers** (clean professional look)
9. **Max length:** 1 page (content should not exceed ~5 paragraphs plus address block)
10. **Left-aligned** throughout (no centering, no justified text)

## Validation Checklist

After generating any cover letter DOCX, verify:

- [ ] Opens correctly in Microsoft Word and Google Docs
- [ ] Page size is US Letter (8.5" × 11")
- [ ] Margins are 1" top/bottom, 1.25" left/right
- [ ] Name line is Calibri 14pt Bold in blue (#365F91)
- [ ] All body text is Cambria 11pt black
- [ ] No double-spacing between lines within paragraphs
- [ ] Clean single blank-line gaps between sections
- [ ] Smart quotes (curly), not straight quotes
- [ ] Fits on 1 page
- [ ] No headers, footers, or page numbers
